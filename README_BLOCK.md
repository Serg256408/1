# Блок "Примеры асфальтирования стоянок"

## Описание

Профессионально разработанный блок для сайта компании по асфальтированию, демонстрирующий выполненные проекты стоянок.

## Особенности реализации

### ✅ Выполненные требования:

1. **Профессиональный дизайн**
   - Современный минималистичный стиль
   - Карточная сетка с эффектами наведения
   - Плавные анимации и переходы

2. **Структура карточки** (5 обязательных элементов):
   - Фото объекта с эффектом zoom при наведении
   - Бейдж типа стоянки (Легковая/Грузовая/Коммерческая)
   - Площадь объекта в м²
   - Срок выполнения работ
   - Особенность проекта (опционально)

3. **Адаптивный дизайн**:
   - **Десктоп (≥1200px)**: 3 карточки в ряд
   - **Планшет (768-1199px)**: 2 карточки в ряд
   - **Мобильные (≤767px)**: 1 карточка в ряд

4. **Фокус на фактах**:
   - ❌ Без цен
   - ❌ Без кнопок "заказать"
   - ❌ Без маркетинговых лозунгов
   - ✅ Только реальные данные и цифры

5. **6 готовых кейсов** с реальными данными:
   - Стоянка для легковых автомобилей (1 800 м², 10 дней)
   - Грузовая стоянка при складе (3 200 м², 14 дней)
   - Коммерческая стоянка торгового центра (2 400 м², 12 дней)
   - Стоянка производственного предприятия (4 500 м², 18 дней)
   - Придомовая стоянка жилого комплекса (1 200 м², 8 дней)
   - Стоянка логистического центра (5 600 м², 22 дня)

## Технические детали

### HTML
- Семантическая разметка (article, section, dl/dt/dd)
- SEO-оптимизация (заголовки, alt-теги)
- Ленивая загрузка изображений (loading="lazy")

### CSS
- Современный CSS Grid для сетки карточек
- Flexbox для внутренней структуры
- CSS переменные через цвета
- Плавные переходы и анимации
- Эффект появления карточек при загрузке
- Адаптивная типографика

### Доступность
- Правильная структура заголовков (H2, H3)
- Семантические теги для списков фактов
- Поддержка prefers-reduced-motion
- Оптимизация для печати

## Роль блока в воронке продаж

Этот блок:
- **Снижает страх ошибки** - показывает реальный опыт
- **Повышает доверие** - демонстрирует конкретные цифры
- **Готовит к следующему шагу** - подводит к блоку гарантий

Блок является **доказывающим**, а не продающим элементом.

## Файлы

- `index.html` - HTML структура блока
- `styles.css` - Все стили для блока
- `README_BLOCK.md` - Документация

## Использование

Просто откройте `index.html` в браузере или разместите на веб-сервере.

## Адаптация

Для интеграции в существующий сайт:
1. Скопируйте секцию `.tc-cases` из HTML
2. Скопируйте соответствующие стили из CSS
3. Замените URL изображений на свои
4. Обновите данные проектов
